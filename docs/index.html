Geospatial Terrain Importer

Real elevation data,

This project turns digital elevation models - the same GeoTIFF rasters used in GIS, simulation, and mapping - into interactive level-of-detail 3D terrain. Below is real satellite-derived elevation, processed by the project's own pipeline and rendered live in your browser.

Drag to orbit · scroll to zoom · built with WebGL

About this project

Geospatial Terrain Importer ingests real-world elevation data and turns it into interactive, level-of-detail 3D terrain. It ships as a C++/Qt/OpenGL desktop application plus a headless Python pipeline - and this page is a WebGL showcase of that pipeline's output.

What you're looking at

A digital elevation model (DEM) is a raster where every pixel stores a ground height in metres. The terrain here comes from the public AWS Terrain Tiles open dataset (SRTM/NED-derived). Each location is a real ~59 km-wide patch of the planet.

The pipeline

  1. Ingest a GeoTIFF DEM via GDAL, with bilinear resampling and nodata/NaN sanitisation.
  2. Build a mesh - a triangulated height-field with per-vertex normals and texture coordinates.
  3. Simplify with Quadric Error Metrics (Garland & Heckbert 1997) and boundary preservation.
  4. Generate an LOD pyramid - multiple resolutions selected by camera distance.
  5. Drape a texture - here a hillshade / elevation colormap; in the app, packed satellite tiles.

Engineering

  • Core (C++17): GDAL ingest, QEM simplifier, LOD manager, texture-atlas shelf-packer, OGR/PROJ coordinate transforms (WGS-84 ↔ UTM).
  • Renderer: OpenGL 4.1 + Qt orbit-camera viewport with height-colormap shaders.
  • Python pipeline: headless batch conversion, LOD generation, atlas packing (numpy / GDAL / Pillow).
  • Quality: C++ unit tests + pytest, run on GitHub Actions CI.
  • This demo: Three.js. Meshes are rebuilt in-browser from float32 heightfields the pipeline exported.
